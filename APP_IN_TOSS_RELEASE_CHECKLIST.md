## 제출 전 필수 명령

```bash
node scripts/test-progression.mjs
npm run typecheck
npm run build
npm run export:direct
npm run verify:layout
npm run verify:render
npm run ait:build
```

`npm run ait:build`는 Toss CLI 및 AIT 포맷 패키지의 Node 요구 조건 때문에 Node 24 이상에서 확인하는 것을 권장합니다. `verify:render`는 Chrome/Edge 실행 파일이 없는 WSL/CI 환경에서 skip될 수 있으므로 로그를 확인합니다.

## P0 안정화 확인

- [ ] 광고 2배 보상은 한 결과 화면에서 한 번만 지급된다.
- [ ] 광고 버튼 연타 중 `inFlight` 상태에서 중복 show/reward가 발생하지 않는다.
- [ ] 광고 미지원/로드 실패/일일 cap 초과 시 게임 흐름이 막히지 않는다.
- [ ] delayedCall 기반 추가 스폰/스톰/경보 콜백은 `phase === playing` 및 scene active 상태에서만 실행된다.
- [ ] 채무동결은 같은 위험물에 감속이 중복 누적되지 않고 `frozenUntil` 이후 base speed로 복구된다.
- [ ] visibilitychange listener는 scene shutdown/destroy 시 remove되어 WebView 재진입 누수를 만들지 않는다.
- [ ] Game Center 점수 제출은 사용자 식별 완료 후 실행되고, 실패/대기 상태가 플레이 흐름을 막지 않는다.

## 화면 QA

- [ ] 메인 메뉴: 시작 버튼, 스테이지, 성장, 랭킹 버튼이 작은 화면에서도 겹치지 않아야 합니다.
- [ ] 메인 메뉴: 현금/피해/아슬 범례와 경보 룰이 10초 안에 이해되어야 합니다.
- [ ] 스테이지 맵: 해금/잠김/선택 상태, 목표 점수, 스테이지 기록이 명확해야 합니다.
- [ ] 첫 플레이 온보딩: 드래그, 현금 수집, 회피, 아슬회피, 업그레이드 선택 단계가 차례로 진행되어야 합니다.
- [ ] 성장 관리: 코인, 강화 레벨, 스킨 진행도, 강화/스킨/닫기 버튼이 한 화면 안에서 읽혀야 합니다.
- [ ] 튜토리얼: 핵심 규칙 3개와 시작 버튼이 안전 영역 안에 있어야 합니다.
- [ ] 플레이: 점수, 시간, 콤보, 미션, 상태 문구가 장애물/보상과 겹치지 않아야 합니다.
- [ ] 경보 단계: 50,000점 단위로 경보 단계가 상승하고 속도/스폰/위험 비율이 체감되어야 합니다.
- [ ] 일시정지: 재개, 사운드 설정, 메뉴 버튼과 통계 문구가 겹치지 않아야 합니다.
- [ ] 업그레이드: 3개 카드의 제목/설명/진화 힌트가 긴 문구에서도 잘려 보이지 않아야 합니다.
- [ ] 게임오버: 큰 점수, 다음 목표, 코인, 미션 결과, 광고 2배/재도전/공유/랭킹 버튼이 안정적으로 보여야 합니다.

## 플레이 QA

- [ ] 현금봉투, 코인, 시간 쿠폰, PAYDAY 오브 수집이 각각 정상 반응해야 합니다.
- [ ] 카드값, 월세, 세금, 구독료 충돌이 체력/속도/흡입 효과와 연결되어야 합니다.
- [ ] 콤보 10단위 시간 보너스가 적용되어야 합니다.
- [ ] 월급각성 게이지와 PAYDAY 오브가 정상 동작해야 합니다.
- [ ] 45초, 30초, 15초 구간 업그레이드 선택 후 라운드가 재개되어야 합니다.
- [ ] `월급자석+캐시백`, `파산보험+가계부 제압`, `상여금 각성+야근수당` 조합에서 EVO 콜아웃이 떠야 합니다.
- [ ] 스테이지 목표 점수 달성 시 다음 스테이지가 해금되고 스테이지 맵에서 선택 가능해야 합니다.
- [ ] 온보딩 완료 후 다음 실행부터는 바로 라운드를 시작할 수 있어야 합니다.
- [ ] 라운드 종료 후 최고 점수, 일일 미션, 코인 보상이 저장되어야 합니다.
- [ ] fallback 캔버스에서도 동일 저장 키의 최고 점수가 유지되어야 합니다.

## Toss WebView QA

- [ ] 햅틱이 사용자 입력 이후에만 호출되어 정책 오류가 없어야 합니다.
- [ ] 이벤트 로그가 화면 전환과 주요 결과에서 중복 과다 없이 남아야 합니다.
- [ ] Game Center 사용자 키 fallback이 브라우저 단독 실행과 Toss WebView에서 모두 안전해야 합니다.
- [ ] 리더보드 열기와 점수 제출 실패가 플레이 흐름을 막지 않아야 합니다.
- [ ] 리뷰 요청은 최고 기록 갱신 후에도 조용히 실패할 수 있어야 합니다.
- [ ] 백그라운드 전환 시 BGM/SFX가 즉시 멈추고 복귀 시 중복 없이 재개되어야 합니다.
- [ ] 광고 중 BGM/SFX가 멈추고 종료 후 정상 복귀해야 합니다.

## 제출 순서

1. Node 24 이상 환경에서 의존성을 깨끗하게 설치합니다.
2. 필수 명령 전체를 실행하고 실패가 없는지 확인합니다.
3. Toss 콘솔에서 Game Center 리더보드와 WebView 타입을 확인합니다.
4. 광고 테스트 ID로 preload/show/reward/close/fail 경로를 확인합니다.
5. 실제 Toss WebView에서 화면 QA와 플레이 QA를 수행합니다.
6. 제출 빌드 산출물 또는 배포 URL을 등록합니다.
