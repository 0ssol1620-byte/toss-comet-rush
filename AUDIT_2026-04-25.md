## 초기 주요 문제

- Windows 환경에서 `tsc`, Vite/Rolldown, esbuild native binary 문제가 있어 빌드 재현성이 낮았습니다.
- 실제 UI가 390 x 844 기준으로 강하게 고정되어 320 x 568급 화면에서 지나치게 작고 답답했습니다.
- 첫 화면 하단에 리더보드, 지갑, 강화, 스킨, footer, 버전 정보가 몰려 텍스트 겹침 위험이 컸습니다.
- `verify:layout`은 정적 검사 위주라 실제 렌더링, 스크린샷, debug state 검증을 보장하지 못했습니다.
- fallback 캔버스가 메인 게임과 다른 저장 키를 사용했습니다.

## v8에서 처리한 사항

- Windows 빌드 실패 시 WSL로 자동 우회하는 `scripts/build.mjs` fallback을 추가했습니다.
- `npm run typecheck`, `npm run verify:render`를 추가해 타입 검사와 실제 브라우저 렌더 검증을 분리했습니다.
- 메인 메뉴를 시작/성장 관리/리더보드 중심으로 줄이고, 메타 기능을 성장 관리 화면으로 이동했습니다.
- Phaser 텍스트 피팅을 HUD, 버튼, 결과, 업그레이드, footer 영역에 적용했습니다.
- debug query 화면을 추가해 `menu`, `growth`, `tutorial`, `playing`, `pause`, `upgrade`, `gameover` 상태를 자동 캡처할 수 있게 했습니다.
- fallback 저장 키를 메인 저장 키와 통일했습니다.
- 하드코딩된 개인 Desktop/OneDrive 경로를 사용자 홈 기반 경로 탐색으로 교체했습니다.

## v9에서 처리한 사항

- 50,000점마다 `월급 경보 단계`가 상승하고 속도/스폰/위험 비율/보상 배율이 같이 변하도록 만들었습니다.
- Stage 1~5 체인을 추가하고 목표 점수 달성 시 다음 스테이지가 해금되도록 저장 구조를 확장했습니다.
- 스테이지별 위험물 가중치와 보상/속도 보정을 추가했습니다.
- `자동환급장`, `채무동결`, `13월의 월급` 업그레이드 조합 진화를 추가했습니다.
- 결과 화면에 스테이지, 경보 단계, 다음 스테이지 해금 상태가 표시되도록 바꿨습니다.
- 바탕화면 export 대상에 `Toss-Comet-Rush-Test.html`을 포함했습니다.

## v10에서 처리한 사항

- 첫 플레이어를 위한 인터랙티브 온보딩을 추가했습니다.
- 온보딩은 드래그 이동, 현금 수집, 회피, 아슬회피, 업그레이드 선택, 실전 시작 단계로 진행됩니다.
- 온보딩 완료 여부를 저장해 다음 실행부터는 바로 라운드를 시작할 수 있게 했습니다.
- 렌더 검증에 `onboarding` 화면을 추가했습니다.
- 같은 WiFi의 폰에서 접속할 수 있도록 `play-direct.html` 서버 테스트를 확인했습니다.

## 남은 출시 전 리스크

- Toss CLI 제출 빌드는 Node 24 이상 환경에서 다시 확인해야 합니다.
- 실제 Toss WebView의 햅틱, 이벤트 로그, 리더보드 제출은 콘솔 권한과 실기기 QA가 필요합니다.
- 현재 저장소는 git repository가 아니라 변경 이력 diff는 로컬 파일 기준으로만 관리됩니다.
