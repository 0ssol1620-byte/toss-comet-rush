# Performance, Audio, and Speed Audit

Date: 2026-04-26

## Findings

- Public web BGM cannot use the NCS test MP3 by default. The NCS file is kept local/test-only because game insertion requires separate licensing, so GitHub Pages falls back to the original procedural BGM.
- The old original BGM was generated by a 170ms Phaser timer that created oscillators and noise buffers during play. That can trigger garbage collection and audible/visual stutter on mobile browsers.
- Frame spikes were applied directly to player movement, falling objects, timers, and spawning. When the browser paused briefly, the next frame could move everything too far and feel like the game suddenly accelerated.
- Difficulty already had a capped 50,000 won alert speed multiplier, but the combined time/stage difficulty path had no explicit top-level clamp.
- Autopilot scanned actors with filter/sort allocations every frame while active.
- Falling item scale animation accumulated `scaleX` over time instead of deriving from a stable base scale.

## Upgrades Applied

- Replaced runtime BGM sequencing with a pre-rendered original EDM loop using one `AudioBufferSourceNode`.
- Kept the local `?ncs=1` test hook, but automatically mutes the original loop when the local NCS file successfully plays.
- Added frame delta clamping at 50ms so browser hiccups no longer create sudden massive movement jumps.
- Added explicit progression caps:
  - score alert speed multiplier: 2.25x
  - effective difficulty: 4.15x
  - falling actor speed: 820 px/s
  - active actors: 38
- Added spawn guard checks for delayed spawns.
- Rewrote autopilot target search to a single allocation-free loop.
- Changed falling actor pulse scaling to derive from `baseScale` every frame, preventing long-session visual drift.
- Bumped the build stamp to `v11` and extended layout verification to assert the new performance/audio safeguards.

## Remaining Production Polish

- For public release, replace local NCS testing with either a licensed commercial track, an in-house track, or a properly licensed game-embedding package.
- Add an in-game audio calibration screen only if user testing still reports low volume after first tap.
- Capture real-device FPS/audio unlock metrics on Android Chrome, iOS Safari, and Toss WebView before paid traffic.
- Consider a quality toggle for older devices if actor count, bloom, and particles still create device-specific frame drops.
